# ‚≠ê **Why this is Method Overloading?**

Method overloading happens when:

* Methods have the **same name**
* But **different parameters** (type, number, or order)

In this program:

| Method                    | Parameters    | Meaning        |
| ------------------------- | ------------- | -------------- |
| `add()`                   | no parameters | fixed addition |
| `add(int, int)`           | 2 parameters  | add 2 numbers  |
| `add(int, int, int, int)` | 4 parameters  | add 4 numbers  |

Java automatically chooses the correct method based on the number of arguments.
